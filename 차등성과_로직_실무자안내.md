## 0. 차등 성과란 — 한 줄 정의

> **3일 단위로 코디 성과를 줄세워 A/B/C 3등급으로 자동 배정하고, 만성 저성과자는 H로 강등하는 시스템.**

- 회차 = **3일** (DB 분배일 기준)
- 평가 풀 = 뇌새김영어 **W·E·A 그룹** 전원 (통합 1개 풀)
- 단일 기준 지표 = **취소반영 계약률**

---

## 1. 개편 핵심 — 무엇이 바뀌었나

| 항목 | 이전 (N그룹) | 차등 개편 |
|---|---|---|
| 등급 명칭 | N_A / N_B / N_C / H | **A / B / C / H** |
| 평가 주기 | 월간 | **3일 회차** |
| 평가 대상 | A그룹만 | **W + E + A 통합 풀** |
| 취소 반영 | 분기 취소율 × 계약수 (40% 상한) | **고객키 매칭 — 회차 반품 직접 차감** |
| H 진입 임계 | 4연속 C → H | **5회차째 C → H** (4연속 후 5번째) |
| 근태 강등 | 없음 | **회차 내 근태이슈 ≥ 2건 → 한 단계 강등** |
| 정당 사유 | 없음 | **회차 3일 전체 휴가/연차 → 직전 회차 유지** |
| 신입 보호 | 입사월+1개월 말일 | **입사 +60일** |
| 프로모션 | 없음 | **B/C 1위 → 한 단계 승급** |

## 2. 등급 체계

| 등급 | 성격 | 색상 | 진입 경로 |
|---|---|---|---|
| **A** | 상위 — 고성과 | 🟨 골드 `#F5B400` | 자동 3분할 상위 / B 1위 프로모션 |
| **B** | 중위 — 표준 | 🟩 그린 `#4CAF7E` | 자동 3분할 중위 / C 1위 프로모션 / 신입 보호 / 근태 A→B |
| **C** | 하위 — 저성과 | 🟥 레드 `#DC3545` | 자동 3분할 하위 / 근태 B→C / DB 0건 강제 / H 탈출 |
| **H** | 페널티 — 만성 저성과 | 🟦 블루 `#4472C4` | C 5회차째 (4연속 후 5번째) |

> 🟦 **H등급은 자동 배정으로는 5회차째에만 진입**합니다. 한 회차 평가만으로는 H가 될 수 없습니다.

### CMS 그룹 매핑 (참고용 표시)

명단 파일의 CMS 그룹 표기는 차등 등급의 자연스러운 의미 매핑일 뿐, 실제 등급 산정과는 무관합니다.

| CMS 그룹 | 대응 등급 |
|---|---|
| W그룹 | A 등급 |
| E그룹 | B 등급 |
| A그룹 | C 등급 |

## 3. 입력 파일

### 3-1. 매 회차 업로드 (4종)

| 종류 | 파일명 예시 | 보관 정책 | 용도 |
|---|---|---|---|
| **DB수** | `WM_고객관리_YYYY-MM-DD.xlsx` | 누적 | 회차 분배량 |
| **계약수** | `WM_고객관리_YYYY-MM-DD.xlsx` | 누적 | 회차 실계약 |
| **취소수** | `신청자관리_YYYY-MM-DD.xlsx` | 최신 1건 | 반품일 + 고객키 매칭 |
| **명단** | `WM_근태보고_그룹_YYYYMMDD.xls` | 최신 1건 | 멤버 추출 + 일자별 근태 |

### 3-2. 초기등급 (1회성)

- 시스템 도입 시점에 관리자가 **A/B/C 초기 등급을 일괄 등록**.
- 템플릿 다운로드 → `이름`, `등급` 두 컬럼만 입력.
- **H는 초기 등급으로 사용 불가** (자동 산정 결과로만 부여).
- 등록 후 회차 산정의 streak 카운트 출발점이 됨.

### 3-3. 회차 기간 결정

- 시작/종료 = **DB 파일 분배일의 min / max**.
- 시트명 = **분배일 max의 YYMMDD** (예: `2026-04-23` → 시트명 `260423`).

## 4. 핵심 지표 — 취소반영 계약률

```
DB수            = 회차 내 분배 (주말·공휴일·재콜·talkis·관리자 행 제외)
실계약수        = 회차 내 '계약' + '취소' 모두 카운트
취소수          = 반품일이 회차 내 + 매칭 계약의 오더일이 달력상 3개월 이내
                 (주말·공휴일 반품, 매칭 실패, 3개월 외 계약은 제외)

취소반영 계약수 = 실계약수 − 취소수            ← 음수 가능
취소반영 계약률 = 취소반영 계약수 / DB수        ← 등급 배정 단일 기준
```

### 계약 vs 취소 분류 (고객상태)

| 분류 | 조건 |
|---|---|
| **계약** | 배송대기 / 배송중 / 자동구매완료 / 구매완료 / AS 포함 |
| **취소** | 반품입고 / 승인요청 / 입고대기 / 환불요청 / 환불완료 (단 `최종통화시간 == 오더일`이면 시스템 오류로 제외) |
| **제외** | 결제보류 / 상담취소 / 상담요청 / 상담대기 / 결제요청 등 |

> 📌 **취소수는 별도 파일(`신청자관리`)** 에서 산출합니다. 고객키로 계약 파일과 매칭되어야만 카운트됩니다.

## 5. 등급 산정 흐름 (전체)

```
┌─────────────────────────────────────────────────────────────┐
│ ① 기본 3분할 (취소반영 계약률 desc) → A 1/3 · B 1/3 · C 1/3 │
└─────────────────────────────────────────────────────────────┘
                            │
                            ▼
┌─────────────────────────────────────────────────────────────┐
│ ② 정당 사유 적용 → 직전 회차 등급/streak 유지 (이후 단계 skip)│
└─────────────────────────────────────────────────────────────┘
                            │
                            ▼
┌─────────────────────────────────────────────────────────────┐
│ ③ H 탈출 → 직전 H + 이번 실계약 ≥ 2건 → C, streak 0 리셋    │
└─────────────────────────────────────────────────────────────┘
                            │
                            ▼
┌─────────────────────────────────────────────────────────────┐
│ ④ Streak 카운트 → C 연속 5회차째에 H 진입                   │
└─────────────────────────────────────────────────────────────┘
                            │
                            ▼
┌─────────────────────────────────────────────────────────────┐
│ ⑤ 근태 강등 → 회차 내 근태이슈 ≥ 2건이면 한 단계 ↓           │
└─────────────────────────────────────────────────────────────┘
                            │
                            ▼
┌─────────────────────────────────────────────────────────────┐
│ ⑥ DB 0건 → C 강제                                            │
└─────────────────────────────────────────────────────────────┘
                            │
                            ▼
┌─────────────────────────────────────────────────────────────┐
│ ⑦ 신입 보호 → 입사 +60일 이내 + C/H → B                      │
└─────────────────────────────────────────────────────────────┘
                            │
                            ▼
┌─────────────────────────────────────────────────────────────┐
│ ⑧ 수동 그룹 베이스라인 적용                                  │
└─────────────────────────────────────────────────────────────┘
                            │
                            ▼
┌─────────────────────────────────────────────────────────────┐
│ ⑨ 프로모션 → B/C 각 1위(실계약수) → 한 단계 ↑                │
└─────────────────────────────────────────────────────────────┘
                            │
                            ▼
                  최종 등급 + 커트라인 표시
```

> ⚠️ 단계는 **순서대로 덮어쓰기**됩니다. 같은 인원에 여러 룰이 해당되면 뒤 단계가 우선합니다.

### 기본 3분할의 동점자 보정

같은 취소반영 계약률(소수 4자리 반올림)은 **절대 다른 등급으로 갈리지 않습니다**.
- A/B 또는 B/C 경계에서 같은 값이 걸리면 → 커트라인을 한 칸 아래로 밀어 동률자를 위 등급에 포함.

## 6. 특수 규칙 상세

### 6-1. 정당 사유 ★ 직전 회차 그대로 유지

**조건**: 회차 3일 전체 출근 0일 + 정당사유 ≥ 1일

**정당사유 인정**: 연차 / 휴가 / 공가 / 경조 / 대체휴무 / 오전 반차 / 오후 반차

**처리**:
- 등급 = **직전 회차와 동일**
- streak = **직전 회차 그대로** (증가/리셋 안 됨)
- 이후 단계 (H 탈출, 근태 강등, 프로모션 등) **모두 skip**
- 표·엑셀에 `정당사유` 뱃지 표시

> 📌 3일 중 **하루라도 출근**했으면 정당사유로 보지 않고 정상 성과 평가합니다.

---

### 6-2. H 탈출 / H 진입

#### H 탈출 (구제)
**조건**: 직전 회차 H + 이번 회차 **실계약 ≥ 2건**

**처리**:
- 등급 → **C로 강제 복귀**
- streak → **0으로 리셋** (다시 5회차 C가 누적되어야 H 재진입 가능)
- 프로모션 풀에서 **제외** (이번 회차에 추가 승급 불가)

#### H 진입
**조건**: C 연속 streak ≥ 5 (4회차 연속 C/H 후 **5번째 회차도 C** 인 경우)

**처리**:
- 등급 → **H로 강등**
- 다음 회차에 실계약 ≥ 2건 달성 시 H 탈출

> 📌 streak는 **C 또는 H 등급으로 보낸 회차 수**를 누적 카운트합니다. 정당사유 회차는 카운트에서 **제외**됩니다.

---

### 6-3. 근태 강등 ★ 회차 내 ≥ 2건

**조건**: 회차 내 근태이슈가 **2건 이상**

**근태이슈로 카운트되는 상태**: 결근 / 오전 결근 / 오후 결근 / 조퇴 / 지각 / 지각 10분 / 지각 20분 / 지각 20분 초과

**처리**:
- A → **B**
- B → **C**
- C → **C 유지** (더 떨어지지 않음)
- H → H 유지 (해당 없음)

> 📌 근태 1회는 표시만 되고 강등은 없습니다 (다음 회차에 1회 더 발생 시 누적이 아닌, 회차당 독립 카운트).

---

### 6-4. DB 0건 → C 강제

**조건**: 회차 내 DB 분배 **0건** + 정당사유 아님

**처리**:
- 등급 → **C 강제 배정**
- 분배 자체를 받지 못한 경우이므로 정상 평가 불가하기에 하위로 처리.

---

### 6-5. 신입 보호 ★ 입사 +60일

**조건**: 입사일 + 60일 이내 + 자동 산정 결과가 **C 또는 H**

**처리**:
- 등급 → **B로 상향** (`isProtected` 플래그 부여)
- streak → **0**
- 표·엑셀에서 **B 등급 최하단**에 정렬되어 일반 B와 시각 구분
- 비고 컬럼에 `신규입사자` 표시

> 📌 신입이 자동으로 A/B를 받았다면 보호는 **적용되지 않습니다** (하향 방지 전용, 상향은 자유).
> 📌 보호 기간 중 **A는 가능**합니다. 성적이 좋으면 정상적으로 A 배정.

---

### 6-6. 프로모션 (승급)

**조건**: B 등급 1위 → A · C 등급 1위 → B

**1위 판정 기준**:
1. 실계약수 (총계약수)
2. 동률 시 취소반영 계약률

**프로모션 풀 제외 대상**:
- 정당사유 인원
- H 탈출자 (이번 회차에 H에서 C로 복귀한 사람)
- 신입 보호자
- DB 0건 강제 인원
- 회차 근태이슈 ≥ 2건

> 📌 수동 그룹은 풀에 **포함**됩니다 (수동 베이스라인 + 프로모션 동시 적용).
> 📌 프로모션이 적용된 인원은 비고 컬럼에 `실계약수 프로모션 1등` 표시.
> 📌 1위 후보가 모두 제외 대상이면 차순위 자동 적용.

#### 프로모션 ON/OFF 토글

대시보드 상단 `프로모션: ON/OFF` 버튼으로 비활성화 가능. 운영 정책상 임시로 끄고 싶을 때 사용.

## 7. 수동 조정 가능 항목

### 7-1. 그룹 수동 변경 (`그룹 수정` 버튼)

- 테이블에서 코디별 등급 드롭다운 선택 → `적용`.
- 변경된 등급은 **베이스라인**으로 저장되어 새로고침/재계산 후에도 유지.
- 자동 룰(정당사유·H 탈출·근태강등 등)은 수동 인원에 대해 **skip**.
- 단, 수동 인원도 **프로모션 풀에는 포함**되어 추가 1단계 승급 가능.
- `[자동복원]` 링크 클릭 시 수동 표시 해제 → 자동 산정으로 복원.

### 7-2. Streak 수동 보정

- 테이블의 streak 배지 클릭 → ±N 입력.
- 예: 정책상 리셋이 필요한 인원은 `-3` 입력 등.
- localStorage `gd_streak_adj`에 누적 보정치 저장.

### 7-3. 프로모션 ON/OFF

- 상단 `프로모션: ON/OFF` 버튼.
- 즉시 재계산되어 결과 갱신.

## 8. 등급 산정 제외 (자동)

명단 파일의 일자별 근태 컬럼에서 **마지막 상태**가 다음일 경우 자동 제외:

| 상태 | 처리 |
|---|---|
| `퇴사` | 평가 풀에서 제외 (영구) |
| `장기공석` | 연속 일수 카운트 후 제외 (30일 이상이면 ⚠ 강조 표시) |

> 📌 제외 인원은 대시보드 상단 노란 배너에 표시됩니다.

## 9. 화면 · 엑셀 표시 규칙

### 9-1. 행 정렬

1. 등급 순: **A → B → C → H**
2. 등급 내: **취소반영 계약률 내림차순**
3. 신입 보호자: **B 등급의 최하단** (B가 없으면 A 최하단)

### 9-2. 커트라인 표시

각 등급 카드 상단에 표시:
- A: `커트라인: XX.X% 이상`
- B: `커트라인: XX.X% 이상`
- C: `XX.X% 미만`

> 📌 커트라인 값은 **신입 보호자 + 프로모션 1위를 제외한** 자연 분포 기준으로 재계산됩니다.

### 9-3. 비고 컬럼 (엑셀/대시보드 공통)

| 표시 | 의미 |
|---|---|
| `실계약수 프로모션 1등` | B/C에서 1위로 한 단계 승급된 인원 |
| `신규입사자` | 입사 +60일 이내 보호 적용 |
| `정당사유` 뱃지 | 회차 3일 전체 휴가 등으로 직전 등급 유지 |
| `⚠ 근태 N` 뱃지 | 회차 내 근태이슈 N건 (2건 이상 빨강) |

### 9-4. 엑셀 구조

- **시트 1개 = 회차 1개** (시트명 = 분배일 max의 YYMMDD)
- 시트 상단: `WM 차등 성과` 타이틀 + 시작일자/종료일자
- 컬럼: 등급 / 코디명 / DB수 / 실계약수 / 실계약률 / 취소수 / 취소반영 계약수 / 취소반영 계약률 / C등급 연속 / 비고
- 음수는 **빨강 볼드**
- 파일명: `(공유) 차등 성과_YY년 M월.xlsx`

## 10. 자주 묻는 질문 / 운영 주의

### Q. C가 5번 연속이 아니라 4번 연속에서 H 가는 거 아닌가요?
A. **5회차째 C일 때 H**입니다. C(1) → C(2) → C(3) → C(4) → C(5) **→ 이 시점에 H**. 4연속까지는 streak 표시만 되고 H 강등은 발생하지 않습니다.

### Q. 신입이 성적이 좋으면 A로 가나요?
A. 네. 신입 보호는 **하향 방지 전용**입니다. C/H로 자동 배정될 때만 B로 끌어올립니다. A/B로 자동 배정되면 그대로 유지됩니다.

### Q. 정당사유와 근태이슈가 같은 회차에 섞이면?
A. 정당사유 회차는 **다른 모든 룰을 skip** 합니다. 근태 강등도 적용되지 않고, 직전 회차 등급이 그대로 유지됩니다.

### Q. 취소수가 회차에 떴는데 등급은 왜 안 떨어지나요?
A. 취소수는 다음 조건을 모두 충족해야 카운트됩니다:
- 반품일이 이번 회차 내
- 매칭되는 계약의 오더일이 **달력상 3개월 이내**
- 반품일이 주말·공휴일 아님
- 고객키로 계약 파일과 매칭됨

3개월보다 오래된 계약의 반품, 주말 반품, 매칭 실패는 모두 제외됩니다.

### Q. 수동 그룹 변경이 다음 회차에도 유지되나요?
A. **유지되지 않습니다.** 수동 그룹은 **해당 회차 내**에서만 베이스라인으로 작동합니다. 다음 회차 산정 시 다시 자동 룰로 산정됩니다.

### Q. 데이터를 잘못 올렸어요. 되돌릴 수 있나요?
A. `파일 관리` 모달에서 잘못된 파일을 삭제 후 재업로드하면 됩니다. DB/계약은 누적이므로 옛 데이터는 그대로 유지됩니다.

### Q. 차등 이력 전체를 초기화하려면?
A. 하단 `이력 초기화` 버튼 → IDB results + streak 보정이 모두 삭제됩니다 (초기등급과 업로드 파일은 유지).

## 11. 1페이지 요약 (PPT 마지막 슬라이드용)

```
3일 회차 · 취소반영 계약률 단일 기준
A 1/3 · B 1/3 · C 1/3 자동 분할

  ↓
정당사유 회차는 직전 등급 그대로 (이후 룰 skip)
직전 H + 실계약 2건 → C 복귀, streak 0 리셋
C 5회차째 → H 강등
근태이슈 2건 이상 → 한 단계 강등 (C는 유지)
DB 0건 → C 강제
입사 60일 이내 + C/H → B 보호
B/C 1위(실계약수) → 한 단계 승급
  ↓
A · B · C · H 최종 배정
```

**기억할 숫자 5개**:
- **3일** 회차
- **5회차째** C → H
- **2건** — H 탈출 / 근태 강등 임계치
- **60일** 신입 보호
- **3개월** 취소 매칭 유효 기간

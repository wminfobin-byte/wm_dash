## 0. 차등 성과란 — 한 줄 정의

> **3일 단위로 코디 성과를 줄세워 A/B/C 3등급으로 자동 배정하고, 성과 0인 사람은 D, 만성 저성과자는 H로 강등하는 시스템.**

- 회차 = **3일** (DB 분배일 기준)
- 평가 풀 = 뇌새김영어 **W·E·A 그룹** 전원 (통합 1개 풀)
- 단일 기준 지표 = **취소반영 계약률** (단, 취소반영 계약수가 0 이하면 D 자동)

---

## 1. 시스템 핵심 원칙

차등 성과 시스템을 떠받치는 7가지 기둥입니다.

| # | 원칙 | 설명 |
|---|---|---|
| 1 | **3일 회차** | DB 분배일 기준 3일을 한 회차로 묶어 등급 산정 |
| 2 | **5등급 체계** | A · B · C · D · H — 위에서 아래로 |
| 3 | **단일 줄세움 지표** | 취소반영 계약률 (취소수까지 차감한 후의 계약률) |
| 4 | **D 자동 분류** | 취소반영 계약수 ≤ 0이면 즉시 D, 나머지만 A/B/C 1/3 분할 |
| 5 | **만성 누적 강등** | D 4회차째에 H로 강등 |
| 6 | **회차 결근 페널티** | 회차 내 근태이슈 ≥ 2건이면 한 단계 강등 |
| 7 | **신입 보호** | 입사 +60일 이내 D/H → C 상향 (D·H 페널티 면제) |

## 2. 등급 체계

| 등급 | 성격 | 색상 | 진입 경로 |
|---|---|---|---|
| **A** | 상위 — 고성과 | 🟨 골드 `#F5B400` | 자동 3분할 상위 / B 1위 프로모션 |
| **B** | 중위 — 표준 | 🟩 그린 `#4CAF7E` | 자동 3분할 중위 / C 1위 프로모션 / 신입 보호 / 근태 A→B |
| **C** | 하위 — 저성과 | 🟧 오렌지 `#F0920E` | 자동 3분할 하위 / 근태 B→C / H 탈출 |
| **D** | 0 이하 — 무성과 | 🟦 블루 `#4472C4` | 취소반영 계약수 ≤ 0 (자동) |
| **H** | 페널티 — 만성 무성과 | ⬛ 그레이 `#4B5563` | D 4회차째 (3연속 후 4번째) |

> 🟦 **H등급은 자동 배정으로는 4회차째에만 진입**합니다. 한 회차 평가만으로는 H가 될 수 없습니다.
> 🟥 **D는 자동 산정 결과 전용**입니다. 초기등급 일괄 등록에서 D/H는 거부됩니다.

### CMS 그룹 매핑 (참고용 표시)

CMS는 차등 등급 명칭으로의 일원화가 즉시 반영되지 않으므로, 당분간은 다음과 같이 매핑됩니다. 명단 파일의 CMS 그룹 표기는 시각적 매핑이며, 실제 등급 산정과는 무관합니다.

| CMS 그룹 | 대응 차등 등급 |
|---|---|
| W그룹 | A 등급 |
| E그룹 | B 등급 |
| A그룹 | C 등급 |
| B그룹 | D 등급 |
| H그룹 | H 등급 |

> 📌 **차등 대상 풀**: 위 5개 CMS 그룹(W·E·A·B·H) 코디 전원이 동일한 풀에서 평가됩니다.

## 3. 입력 파일

### 3-1. 매 회차 업로드 (4종)

| 종류 | 파일명 예시 | 보관 정책 | 용도 |
|---|---|---|---|
| **DB수** | `WM_고객관리_YYYY-MM-DD.xlsx` | 누적 | 회차 분배량 |
| **계약수** | `WM_고객관리_YYYY-MM-DD.xlsx` | 누적 | 회차 실계약 |
| **취소수** | `신청자관리_YYYY-MM-DD.xlsx` | 최신 1건 | 반품일 + 고객키 매칭 |
| **명단** | `WM_근태보고_그룹_YYYYMMDD.xls` | 최신 1건 | 멤버 추출 + 일자별 근태 |

### 3-2. 초기등급 (1회성)

시스템 도입 시점에 한 번만 등록. 두 가지 방식 중 하나를 선택합니다.

#### 방식 1. 이름+등급 엑셀 직접 등록
- 템플릿 다운로드 → `이름`, `등급` 두 컬럼만 입력 → 업로드
- **D/H는 초기 등급으로 사용 불가** (자동 산정 결과로만 부여)

#### 방식 2. 4월 실적으로 자동 산정 (★ 5월 시작 정책 기준)
- 4월 한 달치 **DB · 계약 · 명단** 3종 파일 업로드
- 시스템이 실계약률(취소 미반영) 기준으로 줄세움 → 1/3씩 **A · B · C** 자동 부여
- **D는 부여하지 않음** (초기 산정 시점에는 D 등급 없음, 실적 0인 사람도 C 최하단)
- 시트명 `260430` 고정, 기간 2026-04-01 ~ 2026-04-30로 IDB 저장
- 등록 후 회차 산정의 streak 카운트 출발점이 됨

### 3-3. 회차 기간 결정

- 시작/종료 = **DB 파일 분배일의 min / max**.
- 시트명 = **분배일 max의 YYMMDD** (예: `2026-04-23` → 시트명 `260423`).

## 4. 핵심 지표 — 취소반영 계약수/계약률

```
DB수            = 회차 내 분배 (주말·공휴일·재콜·talkis·관리자 행 제외)
실계약수        = 회차 내 '계약' + '취소' 모두 카운트
취소수          = 반품일이 회차 내 + 매칭 계약의 오더일이 달력상 3개월 이내
                 (주말·공휴일 반품, 매칭 실패, 3개월 외 계약은 제외)

취소반영 계약수 = 실계약수 − 취소수            ← 0 이하면 D 등급 자동
취소반영 계약률 = 취소반영 계약수 / DB수        ← A/B/C 줄세움 기준
```

### 등급 1차 분류 로직

```
취소반영 계약수 ≤ 0  →  D 등급 (즉시 분류)
취소반영 계약수 > 0  →  취소반영 계약률 desc 정렬 → 1/3씩 A/B/C 분할
```

> 📌 D는 "성과 0 또는 마이너스" 인원입니다. DB가 0건이거나, 실계약 0건이거나, 취소가 실계약 이상이면 자동 D.

### 계약 vs 취소 분류 (고객상태)

| 분류 | 조건 |
|---|---|
| **계약** | 배송대기 / 배송중 / 자동구매완료 / 구매완료 / AS 포함 |
| **취소** | 반품입고 / 승인요청 / 입고대기 / 환불요청 / 환불완료 (단 `최종통화시간 == 오더일`이면 시스템 오류로 제외) |
| **제외** | 결제보류 / 상담취소 / 상담요청 / 상담대기 / 결제요청 등 |

> 📌 **취소수는 별도 파일(`신청자관리`)** 에서 산출합니다. 고객키로 계약 파일과 매칭되어야만 카운트됩니다.

## 5. 등급 산정 흐름 (전체)

```
┌─────────────────────────────────────────────────────────────┐
│ ① 1차 분류                                                   │
│   취소반영 계약수 ≤ 0  → D                                   │
│   나머지 → 취소반영 계약률 desc 1/3 분할 (A · B · C)         │
└─────────────────────────────────────────────────────────────┘
                            │
                            ▼
┌─────────────────────────────────────────────────────────────┐
│ ② 정당 사유 → 직전 회차 등급/streak 유지 (이후 단계 skip)    │
│   조건: 회차 모든 일자가 정당사유로만 채워진 경우만         │
└─────────────────────────────────────────────────────────────┘
                            │
                            ▼
┌─────────────────────────────────────────────────────────────┐
│ ③ H 탈출 → 직전 H + 이번 실계약 ≥ 2건 → C, streak 0 리셋    │
└─────────────────────────────────────────────────────────────┘
                            │
                            ▼
┌─────────────────────────────────────────────────────────────┐
│ ④ 근태 강등 → 회차 내 근태이슈 ≥ 2건이면 한 단계 ↓           │
│   A→B, B→C, C→D, D 유지, H 해당 없음                        │
│   (Streak 계산보다 먼저 — 근태로 D 된 사람도 D 누적에 포함)│
└─────────────────────────────────────────────────────────────┘
                            │
                            ▼
┌─────────────────────────────────────────────────────────────┐
│ ⑤ Streak 카운트 → D 3연속 후 4번째 D → H 진입               │
└─────────────────────────────────────────────────────────────┘
                            │
                            ▼
┌─────────────────────────────────────────────────────────────┐
│ ⑥ 신입 보호 → 입사 +60일 이내 + D/H → C                      │
└─────────────────────────────────────────────────────────────┘
                            │
                            ▼
┌─────────────────────────────────────────────────────────────┐
│ ⑦ 수동 그룹 베이스라인 적용                                  │
└─────────────────────────────────────────────────────────────┘
                            │
                            ▼
┌─────────────────────────────────────────────────────────────┐
│ ⑧ 프로모션 → B/C 각 1위(실계약수) → 한 단계 ↑               │
│   (D는 프로모션 대상에서 제외)                               │
└─────────────────────────────────────────────────────────────┘
                            │
                            ▼
                  최종 등급 + 커트라인 표시
```

> ⚠️ 단계는 **순서대로 덮어쓰기**됩니다. 같은 인원에 여러 룰이 해당되면 뒤 단계가 우선합니다.

### 1차 분류의 동점자 보정

같은 취소반영 계약률(소수 4자리 반올림)은 **절대 다른 등급으로 갈리지 않습니다**.
- A/B 또는 B/C 경계에서 같은 값이 걸리면 → 커트라인을 한 칸 아래로 밀어 동률자를 위 등급에 포함.

## 6. 특수 규칙 상세

### 6-1. 정당 사유 ★ 직전 회차 그대로 유지

**조건**: **회차 모든 일자가 정당사유로 처리된 경우만**
- 출근이 1일이라도 있으면 정상 평가 (정당사유 X)
- 결근·지각 등 근태이슈가 1일이라도 섞이면 근태이슈 우선 적용 (정당사유 X, 정상 평가)

**정당사유 인정**: 연차 / 휴가 / 공가 / 경조 / 대체휴무 / 오전 반차 / 오후 반차

**처리**:
- 등급 = **직전 회차와 동일**
- streak = **직전 회차 그대로** (증가/리셋 안 됨)
- 이후 단계 (H 탈출, 근태 강등, 프로모션 등) **모두 skip**
- 표·엑셀에 `정당사유` 뱃지 표시

### 케이스 정리

| 회차 3일 구성 | 처리 |
|---|---|
| 3일 모두 연차/휴가/공가/경조 | 정당사유 회차 → 직전 등급 유지 |
| 1~2일 정당사유 + 나머지 출근 | 정상 평가 (성과 집계 포함) |
| 1일 정당사유 + 2일 결근 | 정상 평가 + 근태 2건 강등 |
| 2일 정당사유 + 1일 결근 | 정상 평가 + 근태 1건 (강등 X, 표시만) |
| 3일 모두 결근 | 정상 평가 + 근태 3건 강등 |

---

### 6-2. H 진입 / H 탈출

#### H 진입
**조건**: D 등급으로 **4회차 연속** (3연속 후 4번째 회차도 D)

**Streak 표시**:
- 1회: 🟧 노란 (s1)
- 2회: 🟧 주황 (s2)
- **3회: 🟥 빨강 (s3) — 다음 D면 H 진입!!**
- 4회+: 이미 H

#### H 탈출 (구제)
**조건**: 직전 회차 H + 이번 회차 **실계약 ≥ 2건**

**처리**:
- 등급 → **C로 강제 복귀**
- streak → **0으로 리셋** (다시 4회차 D가 누적되어야 H 재진입 가능)
- 프로모션 풀에서 **제외** (이번 회차에 추가 승급 불가)

> 📌 streak는 **D 또는 H 등급으로 보낸 회차 수**를 누적 카운트합니다. 정당사유 회차는 카운트에서 **제외**됩니다.

---

### 6-3. 근태 강등 ★ 회차 내 ≥ 2건

**조건**: 회차 내 근태이슈가 **2건 이상**

**근태이슈로 카운트되는 상태**: 결근 / 오전 결근 / 오후 결근 / 조퇴 / 지각 / 지각 10분 / 지각 20분 / 지각 20분 초과

**처리**:
- A → **B**
- B → **C**
- C → **D** (계약 1건 이상이라도 강등됨)
- D → **D 유지** (더 떨어지지 않음)
- H → 해당 없음

> 📌 근태 1회는 표시만 되고 강등은 없습니다 (다음 회차에 1회 더 발생 시 누적이 아닌, 회차당 독립 카운트).
> 📌 **Streak/H 진입 계산보다 먼저 적용**됩니다 — 근태로 D가 된 인원도 D 누적 카운트에 포함됩니다.

---

### 6-4. 신입 보호 ★ 입사 +60일

**조건**: 입사일 + 60일 이내 + 자동 산정 결과가 **D 또는 H**

**처리**:
- 등급 → **C로 상향** (`isProtected` 플래그 부여)
- streak → **0** (D 누적 카운트 리셋)
- 표·엑셀에서 **C 등급 최하단**에 정렬되어 일반 C와 시각 구분
- 비고 컬럼에 `신규입사자` 표시

> 📌 신입이 자동으로 A/B/C를 받았다면 보호는 **적용되지 않습니다** (D·H 면제 전용).
> 📌 즉 신입은 보호 기간 동안 **절대 D·H로 떨어지지 않으며**, 취소반영 계약수가 0 이하이거나 D 누적이 차도 모두 C로 보호됩니다.
> 📌 성적이 좋으면 A까지도 정상 배정 가능 (상향은 자유).

---

### 6-5. 프로모션 (승급)

**조건**: B 등급 1위 → A · C 등급 1위 → B · **D는 프로모션 대상에서 제외**

**1위 판정 기준**:
1. 실계약수 (총계약수)
2. 동률 시 취소반영 계약률

**프로모션 풀 제외 대상**:
- 정당사유 인원
- H 탈출자 (이번 회차에 H에서 C로 복귀한 사람)
- 신입 보호자
- DB 0건 인원
- 회차 근태이슈 ≥ 2건

> 📌 수동 그룹은 풀에 **포함**됩니다 (수동 베이스라인 + 프로모션 동시 적용).
> 📌 프로모션이 적용된 인원은 비고 컬럼에 `실계약수 프로모션 1등` 표시.
> 📌 1위 후보가 모두 제외 대상이면 차순위 자동 적용.

#### 프로모션 ON/OFF 토글

대시보드 상단 `프로모션: ON/OFF` 버튼으로 비활성화 가능. 운영 정책상 임시로 끄고 싶을 때 사용.

## 7. 수동 조정 가능 항목

### 7-1. 그룹 수동 변경 (`그룹 수정` 버튼)

- 테이블에서 코디별 등급 드롭다운 선택 → `적용`. **A/B/C/D/H 모두 선택 가능**.
- 변경된 등급은 **베이스라인**으로 저장되어 새로고침/재계산 후에도 유지.
- 자동 룰(정당사유·H 탈출·근태강등 등)은 수동 인원에 대해 **skip**.
- 단, 수동 인원도 **프로모션 풀에는 포함**되어 추가 1단계 승급 가능.
- `[자동복원]` 링크 클릭 시 수동 표시 해제 → 자동 산정으로 복원.

### 7-2. Streak 수동 보정

- 테이블의 streak 배지 클릭 → ±N 입력.
- 예: 정책상 리셋이 필요한 인원은 `-3` 입력 등.
- localStorage `gd_streak_adj`에 누적 보정치 저장.

### 7-3. 프로모션 ON/OFF

- 상단 `프로모션: ON/OFF` 버튼.
- 즉시 재계산되어 결과 갱신.

## 9. 화면 · 엑셀 표시 규칙

### 9-1. 행 정렬

1. 등급 순: **A → B → C → D → H**
2. 등급 내: **취소반영 계약률 내림차순**
3. 신입 보호자: **C 등급의 최하단** (C가 없으면 B, B도 없으면 A 최하단)

### 9-2. 커트라인 표시

각 등급 카드 상단에 표시:
- A: `커트라인: XX.X% 이상`
- B: `커트라인: XX.X% 이상`
- C: `XX.X% 미만 (계약수>0)`
- D: `취소반영 계약수 ≤ 0`
- H: `D 4회 누적`

> 📌 커트라인 값은 **신입 보호자 + 프로모션 1위를 제외한** 자연 분포 기준으로 재계산됩니다.

### 9-3. 비고 컬럼 (엑셀/대시보드 공통)

| 표시 | 의미 |
|---|---|
| `실계약수 프로모션 1등` | B/C에서 1위로 한 단계 승급된 인원 |
| `신규입사자` | 입사 +60일 이내 보호 적용 |
| `정당사유` 뱃지 | 회차 모든 일자가 정당사유로 처리되어 직전 등급 유지 |
| `⚠ 근태 N` 뱃지 | 회차 내 근태이슈 N건 (2건 이상 빨강) |

### 9-4. 엑셀 구조

- **시트 1개 = 회차 1개** (시트명 = 분배일 max의 YYMMDD)
- 시트 상단: `WM 차등 성과` 타이틀 + 시작일자/종료일자
- 컬럼: 등급 / 코디명 / DB수 / 실계약수 / 실계약률 / 취소수 / 취소반영 계약수 / 취소반영 계약률 / D등급 연속 / 비고
- 음수는 **빨강 볼드**
- 파일명: `(공유) 차등 성과_YY년 M월.xlsx`

## 10. 자주 묻는 질문 / 운영 주의

### Q. D가 3번 연속이 아니라 4번이라야 H 가는 거 맞나요?
A. **D 3회차 연속 후 4번째 D부터 H**입니다. D(1) → D(2) → D(3) → D(4) **→ 이 시점에 H**. 3회까지는 streak 표시만 되고 H 강등은 발생하지 않습니다. 3회차에 빨강 배지로 경고됩니다.

### Q. 신입이 성적이 좋으면 A로 가나요?
A. 네. 신입 보호는 **하향 방지 전용**입니다. C/D/H로 자동 배정될 때만 B로 끌어올립니다. A/B로 자동 배정되면 그대로 유지됩니다.

### Q. 정당사유와 근태이슈가 같은 회차에 섞이면?
A. **회차 모든 일자가 정당사유**여야만 정당사유 회차로 처리됩니다. 결근이 1일이라도 섞이면 정상 평가하고 근태이슈 카운트가 적용됩니다 (2건 이상이면 강등).

### Q. 취소가 실계약보다 많아서 마이너스가 나오면?
A. **취소반영 계약수 ≤ 0이면 D 등급으로 자동 분류**됩니다. C에 들어가지 않습니다.

### Q. DB 0건인 사람은 무슨 등급인가요?
A. DB 0건 → 실계약 0건 → 취소반영 계약수 0 → **자동으로 D 분류**됩니다.

### Q. 취소수가 회차에 떴는데 등급은 왜 안 떨어지나요?
A. 취소수는 다음 조건을 모두 충족해야 카운트됩니다:
- 반품일이 이번 회차 내
- 매칭되는 계약의 오더일이 **달력상 3개월 이내**
- 반품일이 주말·공휴일 아님
- 고객키로 계약 파일과 매칭됨

3개월보다 오래된 계약의 반품, 주말 반품, 매칭 실패는 모두 제외됩니다.

### Q. D 1위는 승급 안 되나요?
A. **D는 프로모션 대상에서 제외**됩니다. D는 "성과 0 이하"라는 정의이므로, 1위라도 0 이하 안에서의 1위는 의미가 없어 승급되지 않습니다.

### Q. 수동 그룹 변경이 다음 회차에도 유지되나요?
A. **유지되지 않습니다.** 수동 그룹은 **해당 회차 내**에서만 베이스라인으로 작동합니다. 다음 회차 산정 시 다시 자동 룰로 산정됩니다.

### Q. 데이터를 잘못 올렸어요. 되돌릴 수 있나요?
A. `파일 관리` 모달에서 잘못된 파일을 삭제 후 재업로드하면 됩니다. DB/계약은 누적이므로 옛 데이터는 그대로 유지됩니다.

### Q. 차등 이력 전체를 초기화하려면?
A. 하단 `이력 초기화` 버튼 → IDB results + streak 보정이 모두 삭제됩니다 (초기등급과 업로드 파일은 유지).

## 11. 1페이지 요약 (PPT 마지막 슬라이드용)

```
3일 회차 · 5등급 (A · B · C · D · H)

  ↓
취소반영 계약수 ≤ 0  →  D 자동
나머지를 취소반영 계약률 desc 1/3 분할 (A · B · C)
  ↓
회차 모든 일자가 정당사유 → 직전 등급 유지 (이후 룰 skip)
직전 H + 실계약 2건 → C 복귀, streak 0 리셋
D 4회차째 → H 강등
근태이슈 2건 이상 → 한 단계 강등 (C 유지, D/H 해당 없음)
입사 60일 이내 + C/D/H → B 보호
B/C 1위(실계약수) → 한 단계 승급 (D는 프로모션 X)
  ↓
A · B · C · D · H 최종 배정
```

**기억할 숫자 5개**:
- **3일** 회차
- **4회차째** D → H
- **2건** — H 탈출 / 근태 강등 임계치
- **60일** 신입 보호
- **3개월** 취소 매칭 유효 기간
